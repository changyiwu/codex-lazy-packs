# 收工同步

保留無關變更，並在遠端操作前取得確認，留下清楚的工作交接。

## 執行流程

1. 讀取適用的 `AGENTS.md`，並檢查 Git 狀態與相關差異。
2. 區分本次工作產生的變更，以及先前存在或與本次無關的變更。
3. 檢查候選檔案是否含有 API 金鑰、權杖、憑證、個人資料或學生姓名。若有發現，在加入暫存區前停止。
4. 更新連結的 Obsidian 駕駛艙，記錄最後動作、目前狀態、下一步與精簡變更紀錄。不要把每日進度寫進 `AGENTS.md`。
5. 只有在長期規則、路徑或專案邊界改變時，才更新 `AGENTS.md`。
6. 提出確切的檔案範圍與提交訊息。只有在使用者明確要求或確認後，才提交或推送。
7. 回報已更新內容、仍留在本機的內容，以及遠端同步結果。

## 安全規則

- 不要加入與本次無關的變更或隱藏憑證。
- 不要強制推送、重設或覆蓋使用者的工作。
- 除非 chezmoi 已設定完成且使用者明確要求，否則不要執行 chezmoi。
- 若 Obsidian 無法使用，提供原定的筆記更新內容，並保持專案檔案不變。
