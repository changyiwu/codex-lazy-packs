# 專案初始化同步

設定專案工作模式，但不要製作第一個班級工具。

## 確認資料

確認專案名稱、用途、資料夾、GitHub 需求、儲存庫公開或私人、GitHub Pages 需求、Obsidian vault 路徑、Firebase 使用狀態與部署目標。只詢問無法安全查得的資料。

## 執行流程

1. 檢查目標資料夾，回報已存在與缺少的項目。
2. 保留既有的 `AGENTS.md`、`README.md`、`.gitignore`、Git 歷史、遠端設定與 Firebase 設定。只加入相容內容。
3. 在 Obsidian vault 根目錄建立或補齊 `<專案名稱>-專案駕駛艙.md`。專案名稱優先採工作資料夾名稱或 repo slug；不要建立 `專案駕駛艙/` 或專案名稱子資料夾。若同名筆記已存在，先讀取並保留既有內容，只合併缺少欄位。建立、補齊、移動或重新命名駕駛艙時，不要更新 `知識庫/log.md` 或其他全域操作紀錄；初始化狀態只寫在駕駛艙本身。
4. 在 `AGENTS.md` 同時記錄 Obsidian vault 絕對路徑與駕駛艙根目錄檔名。
5. 只有在尚未使用 Git 時才初始化。若儲存庫位於雲端同步資料夾，設定 `windows.appendAtomically=false`。
6. 只有在使用者明確要求時才建立或連接 GitHub，並先確認公開或私人及 Pages 需求。
7. 初始化工作區時不要部署，也不要修改 Firebase 安全規則。
8. 驗證路徑、檔案、Git 狀態、遠端網址與 Obsidian 筆記是否可讀。

## 安全規則

- 不要覆蓋既有專案規則或無關變更。
- 不要提交 `.env`、憑證、權杖、密碼、`.codex/` 或 `.claude/`。
- 學生紀錄只使用座號與班級代碼，不要儲存真實姓名。
- 建立 GitHub、變更公開狀態、啟用 Pages、部署、提交與推送都必須由使用者明確選擇。
