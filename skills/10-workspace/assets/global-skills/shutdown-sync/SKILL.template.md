# 收工同步

保留無關變更；安全檢查通過後自動提交並同步 `main`，留下清楚的工作交接。

## 執行流程

1. 讀取適用的 `AGENTS.md`，並檢查 Git 狀態與相關差異。
2. 區分本次工作產生的變更，以及先前存在或與本次無關的變更。
3. 檢查候選檔案是否含有 API 金鑰、權杖、憑證、個人資料或學生姓名。若有發現，在加入暫存區前停止。
4. 準備更新連結的 Obsidian 駕駛艙，記錄最後動作、目前狀態、下一步與精簡變更紀錄。不要把每日進度寫進 `AGENTS.md`。
5. 只有在長期規則、路徑或專案邊界改變時，才更新 `AGENTS.md`。
6. 提出確切的檔案範圍與提交訊息。使用者說「收工」、「結束工作」或「整理並同步」時，視為授權在安全檢查通過後自動 commit／push，不要再詢問一次。若使用者當次明確說不要 push，則只整理與交接。
7. 依下列預設方式自動同步 GitHub：
   - 確認登入帳號為 `changyiwu`，且可寫入的 `origin` 指向 `https://github.com/changyiwu/<repository>.git`。
   - 若原始專案屬於其他帳號，保留為 `upstream`，不要推送到 `upstream`。
   - 先抓取並確認 `origin/main`、工作區與提交範圍，再直接 commit／push 到 `main`。
   - 不建立功能分支或 Pull Request；只有使用者當次明確要求時才使用。
   - 若遇到衝突、未預期的遠端提交或分支保護，停止並回報，不要 force push，也不要自動改走 PR。
8. 推送成功後更新 Obsidian 駕駛艙的提交編號與同步狀態，再回報已更新內容、仍留在本機的內容，以及遠端同步結果。

## 安全規則

- 不要加入與本次無關的變更或隱藏憑證。
- 不要強制推送、重設或覆蓋使用者的工作。
- 「收工」、「結束工作」或「整理並同步」預設包含自動 commit／push；使用者當次明確要求不要 push 時，優先服從。
- 除非 chezmoi 已設定完成且使用者明確要求，否則不要執行 chezmoi。
- 若 Obsidian 無法使用，提供原定的筆記更新內容，並保持專案檔案不變。
